# QSSimulation

## Условие задачи
Система массового обслуживания состоит из обслуживающего аппарата (ОА) и очереди заявок двух типов.

Заявки 1-го типа поступают в "хвост" очереди по случайному закону с интервалом времени Т1, равномерно распределенным от 0 до 5 единиц времени (е.в.). В ОА они поступают из "головы" очереди по одной и обслуживаются также равновероятно за время Т2 от 0 до 4 е.в., после чего покидают систему.

Единственная заявка 2-го типа постоянно обращается в системе, обслуживаясь в ОА равновероятно за время Т3 от 0 до 4 е.в. и возвращаясь в очередь на 4-ю позицию от "головы" (или в конец очереди, если ее длина меньше 3). В начале процесса заявка 2-го типа входит в ОА, оставляя пустую очередь. (Все времена – вещественного типа) Смоделировать процесс обслуживания первых 1000 заявок 1-го типа. Выдавать после обслуживания каждых 100 заявок 1-го типа информацию о текущей и средней длине очереди, количестве вошедших и вышедших заявок и о среднем времени пребывания заявок в очереди. В конце процесса выдать общее время моделирования, время простоя аппарата, количество вошедших в систему и вышедших из нее заявок первого типа и количество обращений заявок второго типа. По требованию пользователя выдать на экран адреса элементов очереди при удалении и добавлении элементов. Проследить, возникает ли при этом фрагментация памяти.
## Исходные данные: 
Взаимодействие с программой происходит через меню-интерфейс. Пользователь вводит числа (0-5) для выбора действий программы.

1, чтобы сменить параметры времени заявок

2, чтобы расчитать систему очерьдью-массивом

3, чтобы расчитать систему очерьдью-списком

4, чтобы расчитать погрешность

5, чтобы переключить показ адресов элементов очереди

0, чтобы завершить

Смена параметров заявок происходит через последовательный ввод минимальных и максимальных T1, T2, T3.
## Выходные данные:
При расчёте системы выводится:

- Текущая и средняя длина очереди, количество вошедших и вышедших заявок, а также среднее время нахождения заявки в очереди, для каждой сотой вышедшей заявки первого типа
- Если включен показ адресов выводятся адреса памяти элементов очереди для каждого добавления и исключения элемента очереди
- Результаты расчёта - время моделирования, время простоя, количество вошедших и вышедших заявок первого типа, количество обработанных заявок второго типа
- Затраты памяти и времени на расчёт системы

## Задача, реализуемая программой
Моделирование системы массового обслуживания первых 1000 заявок первого типа.
## Способ обращения к программе
Через терминал (app.exe).
## Возможные аварийные ситуации и ошибки пользователя
Некорректное обращение к меню. исспользование недопустимых символов, неверные временные параметры. Нехватка памяти для инициализации очереди/элемента очереди, переполнение очереди.
